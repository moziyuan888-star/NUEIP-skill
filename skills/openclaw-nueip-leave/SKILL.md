# OpenClaw NUEIP 請假技能

依照以下固定流程執行，除非使用者明確要求變更。

## 1) 開站與登入判斷

1. 使用瀏覽器開啟：`https://cloud.nueip.com/leave_application/personal_leave_application_user`。
2. 偵測是否需要登入（出現登入欄位或被導向登入頁）。
3. 若需要登入且查無可用帳號，詢問並收集：
   - 姓名（帳號別名）
   - 公司代碼
   - 員工編號
   - 密碼
4. 支援多組帳號：
   - 「幫我請假」→ 使用預設帳號。
   - 「幫 XXX 請假」→ 使用姓名 `XXX` 對應帳號。
5. 若找不到對應帳號，先補問帳號資訊再登入。

## 2) 帳號保存規則

1. 將「姓名 → 公司代碼/員工編號/密碼」保存於本地可重用儲存。
2. 之後任務優先套用已儲存資料，避免重複詢問。
3. 若登入失敗，提示使用者更新憑證並覆寫舊資料。
4. 不在回覆中輸出明文密碼。

## 3) 日期與時間解析規範

1. **日期**：接受 `3/7`、`03/07`、`0307` 等寫法，統一正規化為當年度 `YYYY-MM-DD`。
2. **時間**：接受 `9:30`、`09:30`、`0930` 等寫法，統一正規化為 `HH:MM`。
3. 接受 `-` 或 `~` 作為時間區間分隔符。
4. 接受 `/` 分隔同一天的多段時段。
5. 忽略文字中的工時描述（如 `特休3h`、`1H46m`），以實際時間區間為準。
6. 若發現結束時間早於開始時間、格式不完整、或日期無法判讀，先向使用者確認再繼續。

## 4) 單筆請假

當使用者提供單一日期與時段（如 `3/7 9:30-13:00`）時：

1. 點選「請假申請」。
2. 維持「選擇人員」預設值，不調整。
3. 「假別」選擇 **特休(週年制)**。
4. 起日與迄日都輸入同一天，然後點「新增日期」。
5. 在新增出的該日期列，將預設時段改為使用者指定區間：
   - 起始欄位輸入開始時間。
   - 結束欄位輸入結束時間。
6. 原因欄固定輸入：`特休`。
7. 按下「確定」送出。

## 5) 多筆請假

當使用者一次提供多行請假資料時，逐行解析並逐筆填入。

1. 點選「請假申請」。
2. 維持「選擇人員」預設值；假別固定選 **特休(週年制)**。
3. 對每一筆日期：
   - 起日、迄日都輸入同一天，按「新增日期」。
   - 將該日期列預設時段改成指定時段。
4. 若同一天有多個區間（例如 `9:30~9:36/13:00~14:00/17:50~18:30`），在同一日期下依序新增/填入所有區間。
5. 全部日期完成後，原因欄統一填：`特休`，再按「確定」。

## 6) 執行後回報

提交成功後，回報：

1. 使用的帳號別名（姓名）
2. 已提交的日期與時段摘要
3. 是否為單筆或多筆提交

## 7) 例外處理

1. 頁面載入異常或表單提交失敗時，清楚指出失敗步驟並詢問是否重試。
2. 若使用者未指定假別，預設使用特休(週年制)；若需其他假別，先確認再填。
3. 任何一步失敗時，回報失敗原因與需要使用者補充的資訊。
