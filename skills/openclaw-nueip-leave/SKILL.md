# OpenClaw NUEIP 請假技能

依照以下固定流程執行，除非使用者明確要求變更。

## 1) 開站與登入判斷

1. 使用瀏覽器開啟：`https://cloud.nueip.com/leave_application/personal_leave_application_user`。
2. 偵測是否在登入頁或出現登入欄位。
3. 若需要登入，先判斷是否已有可用帳號資料；若沒有，向使用者詢問並收集：
   - 姓名（作為帳號別名）
   - 公司代碼
   - 員工編號
   - 密碼
4. 支援多組帳號。當使用者說「幫我請假」時，優先使用預設帳號；當使用者說「幫 XXX 請假」時，使用姓名 `XXX` 對應帳號。
5. 若找不到對應帳號，立即回到詢問流程補齊資料後再登入。

## 2) 帳號保存規則

1. 將每位使用者的「姓名 → 公司代碼/員工編號/密碼」保存於本地可重用儲存。
2. 後續任務先查詢已儲存資料，避免重複詢問。
3. 若登入失敗，提示使用者更新該帳號資料並覆寫舊值。
4. 僅在執行請假流程必要時使用憑證，不在回覆中外露明文密碼。

## 3) 單筆請假（情況 1）

當使用者提供像 `3/7 9:30-13:00` 的指令時：

1. 點選「請假申請」。
2. 在請假表單中維持「選擇人員」預設值，不調整。
3. 「假別」選擇 **特休(週年制)**。
4. 將日期換算成當年度 ISO 格式：`YYYY-MM-DD`（例如 2026 年的 `3/7` 轉為 `2026-03-07`）。
5. 在起日與迄日都輸入同一天，然後點「新增日期」。
6. 在新增出的該日期列，將預設時段改為使用者指定區間：
   - 起始欄位輸入開始時間（例：`09:30`）
   - 結束欄位輸入結束時間（例：`13:00`）
7. 原因欄固定輸入：`特休`。
8. 按下「確定」送出。

## 4) 多筆請假（情況 2）

當使用者一次提供多行請假資料時，逐行解析並逐筆填入。

1. 點選「請假申請」。
2. 維持「選擇人員」預設值；假別固定選 **特休(週年制)**。
3. 對每一筆日期：
   - 轉為當年度 `YYYY-MM-DD`。
   - 起日、迄日都輸入同一天，按「新增日期」。
   - 將該日期列預設時段改成指定時段。
4. 若同一天有多個區間（例如 `9:30~9:36/13:00~14:00/17:50~18:30`），在同一日期下依序新增/填入所有區間。
5. 忽略文字中的工時描述（如 `特休3h`、`1H46m`），以實際時間區間為準。
6. 全部日期完成後，原因欄統一填：`特休`，再按「確定」。

## 5) 時間解析規範

1. 接受 `9:30`、`09:30`、`0930` 等寫法，統一正規化成 `HH:MM`。
2. 接受 `-` 或 `~` 作為區間分隔符。
3. 接受 `/` 分隔多段時段。
4. 若發現結束時間早於開始時間、格式不完整、或日期無法判讀，先向使用者確認再提交。

## 6) 執行後回報

提交成功後，回報：

1. 使用的帳號別名（姓名）
2. 已提交的日期與時段摘要
3. 是否為單筆或多筆提交

若任何一步失敗，清楚指出失敗步驟與需要使用者補充的資訊。
