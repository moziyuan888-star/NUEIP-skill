# OpenClaw NUEIP 公出技能

依照以下固定流程執行，除非使用者明確要求變更。

## 1) 開站與登入判斷

1. 使用瀏覽器開啟：`https://cloud.nueip.com/inout_record`。
2. 偵測是否需要登入（出現登入欄位或被導向登入頁）。
3. 若需要登入且查無可用帳號，詢問並收集：
   - 姓名（帳號別名）
   - 公司代碼
   - 員工編號
   - 密碼
4. 支援多組帳號：
   - 「幫我公出」→ 使用預設帳號。
   - 「幫 XXX 公出」→ 使用姓名 `XXX` 對應帳號。
5. 若找不到對應帳號，先補問帳號資訊再登入。

## 2) 帳號保存規則

1. 將「姓名 → 公司代碼/員工編號/密碼」保存於本地可重用儲存。
2. 之後任務優先套用已儲存資料，避免重複詢問。
3. 若登入失敗，提示使用者更新憑證並覆寫舊資料。
4. 不在回覆中輸出明文密碼。

## 3) 日期與時間解析規範

1. **日期**：接受 `3/26`、`03/26`、`0326` 等寫法，統一正規化為當年度 `YYYY-MM-DD`。
2. 若未提供日期，預設使用今天日期。
3. **時間**：接受 `16:30`、`1630` 等寫法，統一正規化為 `HH:MM`。
4. 若日期或時間無法判讀，先向使用者確認再繼續。

## 4) 進入新增公出表單

1. 進入頁面後，點「新增」。
2. 若頁面中有「在職狀態 / 在職 / 部門 / 員工 / 加入」區塊，點「加入」。
3. 開始填寫公出資料。

## 5) 欄位填寫規則

1. **日期**
   - 依解析規範填入正規化後的日期。
2. **外出時間（必填）**
   - 這是下拉選單，先展開再選擇「時」與「分」。
3. **返回時間（選填）**
   - 僅在使用者明確提供返回時間時才填。
   - 若未提供，保持空白。
4. **下班不進公司（勾選）**
   - 當使用者語意包含「不回公司」「下班不進公司」時，勾選此選項。
5. **目的地（必填）**
   - 將使用者給的目的或說明原樣填入。

## 6) 提交與回報

1. 確認必要欄位已填（至少日期、外出時間、目的地）。
2. 點擊最下方「確定」提交。
3. 提交後回報：
   - 帳號別名（姓名）
   - 日期
   - 外出時間
   - 是否有返回時間
   - 是否勾選「下班不進公司」
   - 目的地

## 7) 範例對應

使用者說：「今天 16:30 公出 不回公司 去登出班」時，執行：

1. 日期填今天（`YYYY-MM-DD`）。
2. 外出時間選 `16`:`30`。
3. 返回時間不填。
4. 勾選「下班不進公司」。
5. 目的地填 `去登出班`。
6. 按「確定」送出。

## 8) 例外處理

1. 頁面載入異常或表單提交失敗時，清楚指出失敗步驟並詢問是否重試。
2. 若使用者未提供目的地，先詢問再填寫。
3. 任何一步失敗時，回報失敗原因與需要使用者補充的資訊。
