# OpenClaw NUEIP 公出技能

依照以下固定流程執行，除非使用者明確要求變更。

## 1) 開站與登入判斷

1. 使用瀏覽器開啟：`https://cloud.nueip.com/inout_record`。
2. 偵測是否需要登入（登入欄位或登入頁）。
3. 若需要登入且沒有可用帳號，先詢問並收集：
   - 姓名（作為帳號別名）
   - 公司代碼
   - 員工編號
   - 密碼
4. 支援多組帳號。
   - 使用者說「幫我公出」：使用預設帳號。
   - 使用者說「幫 XXX 公出」：使用姓名 `XXX` 對應帳號。
5. 若找不到對應帳號，立即向使用者補問資料再登入。

## 2) 帳號保存規則

1. 將「姓名 → 公司代碼/員工編號/密碼」保存於本地可重用儲存。
2. 之後任務優先讀取已保存帳號，避免重複詢問。
3. 登入失敗時，要求使用者更新該帳號資料並覆寫。
4. 不在回覆中暴露明文密碼。

## 3) 進入新增公出表單

1. 進入頁面後，點「新增」。
2. 若頁面中有「在職狀態 / 在職 / 部門 / 員工 / 加入」區塊，點「加入」。
3. 開始填寫公出資料。

## 4) 欄位填寫規則

1. **日期**
   - 若使用者提供日期，使用使用者指定日期。
   - 若未提供日期，預設使用今天日期（系統當天，格式 `YYYY-MM-DD`）。
   - 將輸入正規化為 `YYYY-MM-DD`（例如 `2026-03-26`、`2026-0303` 都需整理成正確格式）。
2. **外出時間（必填）**
   - 這是下拉選單，先展開再選擇「時」與「分」。
   - 例：`16:30` 需選 `16` 與 `30`。
3. **返回時間（選填）**
   - 僅在使用者明確提供返回時間時才填。
   - 若未提供，保持空白。
4. **下班不進公司（勾選）**
   - 當使用者語意包含「不回公司」「下班不進公司」時，勾選此選項。
5. **目的地**
   - 將使用者給的目的或說明原樣填入（例如：`去登出班`）。

## 5) 提交流程

1. 確認必要欄位已填（至少日期、外出時間、目的地）。
2. 點擊最下方「確定」提交。
3. 提交後回報：
   - 使用帳號別名（姓名）
   - 日期
   - 外出時間
   - 是否有返回時間
   - 是否勾選「下班不進公司」
   - 目的地

## 6) 範例對應

使用者說：「今天 16:30 公出 不回公司 去登出班」時，執行：

1. 日期填今天（`YYYY-MM-DD`）。
2. 外出時間選 `16`:`30`。
3. 返回時間不填。
4. 勾選「下班不進公司」。
5. 目的地填 `去登出班`。
6. 按「確定」送出。
